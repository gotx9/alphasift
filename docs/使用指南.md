## 项目简介

AlphaSift 是一个 A 股自动选股工具，采用**三层漏斗架构**：

```
全市场股票 → L1 硬筛选 → L2 LLM 排序 → L3 后置分析 → 最终候选
```

| 层级 | 功能 | 说明 |
|------|------|------|
| **L1** | 硬筛选 | 基于 YAML 策略规则，代码级过滤 |
| **L2** | LLM 排序 | 可选，使用大模型进行横向比较和语义排序 |
| **L3** | 后置分析 | 可选，scorecard（默认）、DSA、外部 HTTP |

### 核心特点

- **策略即 YAML**：所有选股逻辑通过配置文件定义，无需写代码
- **多数据源**：支持 efinance、akshare_em、em_datacenter，自动降级
- **LLM 可选**：可关闭 LLM 排序，快速筛选
- **评估闭环**：支持 T+N 后验评估

### 1. 系统要求

- Python >= 3.10
- Windows / Linux / macOS

### 2. 创建虚拟环境（推荐）


## 常用命令速查

### 基础命令

```bash
# 列出所有策略
alphasift strategies

# 运行选股（无 LLM，最快）
alphasift screen dual_low --no-llm

# 运行选股（带 LLM 排序）
alphasift screen balanced_alpha

# 查看详细解释
alphasift screen balanced_alpha --explain

# 添加市场上下文
alphasift screen balanced_alpha --context "今日券商板块放量"
```

### 结果保存与评估

```bash
# ⚠️ 默认不保存结果！需显式指定 --save-run
alphasift screen dual_low --no-llm --save-run

# 查看已保存的运行
alphasift runs

# 评估某个运行（T+N 后验）
alphasift evaluate <run_id> --explain

# 带价格路径的评估（计算最大回撤/浮盈）
alphasift evaluate <run_id> --with-price-path --explain

# 批量评估最近 20 次运行
alphasift evaluate-batch --limit 20 --explain
```

### L3 后置分析

```bash
# 默认使用 scorecard（本地评分）
alphasift screen balanced_alpha

# 追加 DSA 深度分析（需配置 DSA_API_URL）
alphasift screen balanced_alpha --post-analyzer dsa

# 关闭 L3 后置分析
alphasift screen balanced_alpha --no-post-analysis
```

### 调试与诊断

```bash
# 项目自检
alphasift audit

# 构建行业热度缓存
alphasift industry-cache --output data/industry_map.csv --explain
```

## 策略说明

### 内置策略一览

| 策略名 | 类型 | 适用场景 |
|--------|------|----------|
| `dual_low` | 价值 | 低 PE + 低 PB，适合保守型价值投资 |
| `quality_value` | 价值 | 估值合理 + 流动性好 + 波动可控 |
| `volume_breakout` | 趋势 | 放量突破关键阻力位 |
| `capital_heat` | 动量 | 资金活跃但未过热 |
| `oversold_reversal` | 反转 | 超跌修复候选 |
| `shrink_pullback` | 趋势 | 均线多头 + 缩量回踩 |
| `balanced_alpha` | 综合 | 多因子均衡策略，LLM 友好 |

### 策略参数

```bash
# 限制输出数量
alphasift screen dual_low --max-output 10

# 关闭 L3 后置分析
alphasift screen dual_low --no-post-analysis

# 使用 DSA 作为 L3 分析器（需配置 DSA_API_URL）
alphasift screen dual_low --post-analyzer dsa
```

### 自定义策略

在 `strategies/` 目录添加 YAML 文件即可。详见 `docs/strategy-guide.md`。

## LLM 配置

### 方式一：极简配置

创建 `.env` 文件：

```env
# 选择一个供应商即可
GEMINI_API_KEY=your_gemini_key
# 或
DEEPSEEK_API_KEY=your_deepseek_key
# 或
OPENAI_API_KEY=your_openai_key
```

### 方式二：指定模型

```env
LITELLM_MODEL=gemini/gemini-2.5-flash
# 或
LITELLM_MODEL=openai/gpt-4o
# 或
LITELLM_MODEL=deepseek/deepseek-chat
```

### 方式三：多渠道配置

```env
LLM_CHANNELS=primary
LLM_PRIMARY_PROTOCOL=openai
LLM_PRIMARY_BASE_URL=https://api.deepseek.com/v1
LLM_PRIMARY_API_KEYS=sk-xxx,sk-yyy
LLM_PRIMARY_MODELS=deepseek-chat,deepseek-reasoner
LITELLM_MODEL=openai/deepseek-chat
```

### 方式四：复用其他项目配置

```bash
alphasift --env-file /path/to/daily_stock_analysis/.env screen balanced_alpha
```

### 重要 LLM 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `LITELLM_MODEL` | 主模型 | `gemini/gemini-2.5-flash` |
| `LLM_RANK_WEIGHT` | LLM 排序权重 | `0.40` |
| `LLM_TEMPERATURE` | 生成温度 | `0.2` |
| `LLM_MAX_CANDIDATES` | 单次最多候选数 | `30` |

## Python API

### 基础用法

```python
from alphasift import screen, list_strategies, evaluate_saved_run

# 列出策略
strategies = list_strategies()
print(strategies)

# 运行选股（无 LLM）
result = screen("dual_low", use_llm=False)

# 遍历结果
for pick in result.picks:
    print(f"{pick.rank}. {pick.code} {pick.name}")
    print(f"   得分: {pick.final_score:.1f}")
    print(f"   价格: {pick.price}")
    print(f"   PE: {pick.pe_ratio}")
```

### 带 LLM 排序

```python
result = screen("balanced_alpha", use_llm=True)

# LLM 输出信息
print(f"市场观点: {result.llm_market_view}")
print(f"选择逻辑: {result.llm_selection_logic}")
print(f"组合风险: {result.llm_portfolio_risk}")

# 候选详情
for pick in result.picks:
    print(f"--- {pick.name} ---")
    print(f"LLM 置信度: {pick.llm_confidence}")
    print(f"LLM 行业: {pick.llm_sector}")
    print(f"LLM 主题: {pick.llm_theme}")
    print(f"投资论点: {pick.llm_thesis}")
    print(f"关注事项: {pick.llm_watch_items}")
    print(f"风险因素: {pick.llm_risks}")
```

### 评估保存的运行

```python
from alphasift import evaluate_saved_run, evaluate_saved_runs

# 评估单个运行
report = evaluate_saved_run("<run_id>", explain=True)
print(f"总收益: {report.total_return_pct:.2f}%")
print(f"胜率: {report.win_rate:.1%}")

# 批量评估
reports = evaluate_saved_runs(limit=10, explain=True)
for r in reports:
    print(f"{r.run_id}: {r.total_return_pct:.2f}%")
```

## 常见问题

### Q0: 运行结果会自动保存吗？

**不会**。AlphaSift 默认不保存运行结果，需要显式使用 `--save-run` 参数：

```bash
# 保存运行结果
alphasift screen dual_low --no-llm --save-run

# 然后可以查看
alphasift runs
```

保存的结果可用于后续 T+N 评估。

### Q1: 周末/节假日无法获取数据？

**原因**：`efinance` 和 `akshare_em` 实时推送接口在非交易时段不可用。

**解决**：系统会自动降级到 `em_datacenter` 数据源，使用最近交易日数据。

### Q2: LLM 排序报错？

检查以下配置：

1. 是否设置了有效的 API Key
2. 模型名称是否正确（如 `gemini/gemini-2.5-flash`）
3. 网络是否能访问 API

快速测试：
```bash
# 不使用 LLM 测试
alphasift screen dual_low --no-llm
```

### Q3: 如何关闭 L3 后置分析？

```bash
alphasift screen dual_low --no-post-analysis
```

### Q4: 输出乱码？

Windows 终端编码问题。尝试：
```bash
chcp 65001
alphasift strategies
```

### Q5: 如何使用本地 Ollama？

```env
OLLAMA_API_BASE=http://localhost:11434
LITELLM_MODEL=ollama/llama3
```

### Q6: DSA 是什么？

DSA（Daily Stock Analysis）是一个可选的外部深度分析服务。AlphaSift 只对最终候选调用 DSA，不参与全市场初筛。

配置：
```env
DSA_API_URL=http://localhost:8000
```

使用：
```bash
alphasift screen dual_low --post-analyzer dsa
```

## 附录：数据源说明

| 数据源 | 接口 | 可用时段 | 特点 |
|--------|------|----------|------|
| `efinance` | push2.eastmoney.com | 交易时段 | 实时推送，最快 |
| `akshare_em` | 82.push2.eastmoney.com | 交易时段 | 备选实时源 |
| `em_datacenter` | data.eastmoney.com | 全时段 | 选股器 API，非交易时段可用 |

降级顺序：`efinance` → `akshare_em` → `em_datacenter`
